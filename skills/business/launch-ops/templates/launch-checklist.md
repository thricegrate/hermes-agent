# Pre-Launch Checklist

Aligned with the Automatic Clients Launch Checklist. Complete every section before going live.

## Deliverables

### Core Product
- [ ] All modules uploaded to Skool/Notion

- [ ] Pinned post created
- [ ] Auto-welcome DM configured

- [ ] All videos play correctly
- [ ] All PDFs/downloads work
- [ ] Module order is logical

### Bonuses
- [ ] All bonus materials created and uploaded
- [ ] Bonus access integrated into main product delivery
- [ ] Each bonus has clear instructions for use

### OTO 1 (One-Time Offer)
- [ ] OTO 1 product/content created and hosted
- [ ] OTO 1 sales page live and proofread
- [ ] OTO 1 payment link configured
- [ ] OTO 1 purchase -> access delivery automation working
- [ ] OTO 1 redirect from main thank-you page working

### OTO 2 (if applicable)
- [ ] OTO 2 product/content created and hosted
- [ ] OTO 2 sales page live and proofread (downsell positioning)
- [ ] OTO 2 payment link configured

### Value Bombs (3-7 pieces)
- [ ] All value bomb content created (from `content-social` Tier 2)
- [ ] Value bomb assets hosted (PDFs, videos, templates)
- [ ] Value bomb email series loaded (from `email-sequence` Step 7)
- [ ] Value bomb social content scheduled (from `content-social` calendar)
- [ ] Distribution timeline mapped: Day 3-14 post-purchase

### Awareness Bridge
- [ ] Awareness bridge email sequence loaded (from `email-sequence` Step 8)
- [ ] Bridge content moves through 5 awareness levels
- [ ] Soft CTAs progress toward call booking
- [ ] Timeline mapped: Day 14-21 post-purchase

### Backend Plan
- [ ] Backend offer defined (DWY/DFY tier from `product-offer`)
- [ ] Pricing and packaging finalized
- [ ] Sales call script prepared (from `high-ticket-closer`)
- [ ] High-ticket nurture sequence loaded (from `email-sequence` Step 9)

### Membership Area / Tech
- [ ] Skool or Notion workspace fully configured
- [ ] All automation triggers tested with test data
- [ ] Payment platform connected and test purchase completed
- [ ] Email platform connected with all segments created

### Backend Deliverables
- [ ] DWY/DFY offer deliverables scoped and ready to fulfill

- [ ] Service delivery timeline established

### Call Booking Page
- [ ] Call booking page live with calendar embedded

- [ ] Availability windows set (3-5 slots/week to start)
- [ ] Buffer time between calls configured (15 min minimum)

### Call Confirmation Page
- [ ] Confirmation page shows after booking

- [ ] Pre-call content link embedded

- [ ] "What to expect" video embedded (optional)

### Call Automations
- [ ] Booking confirmed -> confirmation email sent

- [ ] Sales call script prepared (from `high-ticket-closer`)

- [ ] Red flag screening criteria defined

### Order Form / Checkout
- [ ] High-ticket payment link or invoice process ready
