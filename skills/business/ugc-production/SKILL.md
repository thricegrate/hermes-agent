# UGC Production System (Orchestrator)

Orchestrates AI-generated UGC campaign production from validated concept to scaled output. This skill coordinates the production pipeline across multiple component skills. It does not replace them -- it sequences and manages them.

## Before You Start

**Required inputs:**
- Completed 5-Layer Persona Depth Profile (from `niche-finder` Step 4b)
- Pre-launch economics validated (from `product-offer` Step 0)
- At least 1 validated script concept (from `video-scriptwriter` UGC Cold Traffic framework)
- Quiz funnel live and tracking (from `quiz-funnel`)

**Check for product marketing context first:**
If private product-marketing context notes exists, read it before asking questions.

## The Production Pipeline

```
niche-finder (persona depth)
  -> video-hook (5 UGC hook variations per concept)
    -> video-scriptwriter (6-section UGC scripts)
      -> elevenlabs (speech-to-speech voice production)
        -> jogg-ai (AI avatar video generation)
          -> content-social (distribution across accounts)
            -> quiz-funnel (conversion mechanism)
              -> email-sequence (segmented follow-up)
```

This skill manages the flow through this pipeline, not the work within each step.

## The 4-Axis Variation Generator

Every validated concept contains multiple distinct variations generated by changing 1 element while keeping all others constant.

### Axis 1: Hook Variation (5 per concept)

For every validated script, produce 5 hook variations from the UGC cold-traffic categories (see `video-hook/references/ugc-cold-traffic-hooks.md`):

1. Specificity-Driven Identification
2. Result Specificity
3. Counterintuitive Claim
4. Before-State Immersion
5. Peer Observation

Sections 2-6 of the script remain identical across all hook variations.

### Axis 2: Avatar Variation (4 per concept)

Generate 3-4 avatar variants representing different presentations of the same persona:

- **Avatar A (Aspirational):** Appears at or near the end of the transformation journey
- **Avatar B (Relatable):** Appears in the middle of the journey, closer to the before state
- **Avatar C (Peer Authority):** Looks like a credible practitioner or knowledgeable peer
- **Avatar D (Demographic Variant):** Same general persona, different specific demographic (age, ethnicity, environment)

All 4 avatars read the same script. The test: which visual presentation earns the strongest trust and identification response.

### Axis 3: Length Variation (3 per concept)

- **15-20s:** Hook + core mechanism + CTA only. For retargeting or high-awareness audiences.
- **30-35s:** Hook + abbreviated pain + mechanism + CTA. Standard version.
- **45-55s:** Full 6-section architecture. For cold audiences who need the complete narrative arc.

### Axis 4: Delivery Register (2 per concept)

- **High Energy:** Enthusiasm and excitement in the transformation section. Better for fitness, beauty, lifestyle.
- **Calm Authority:** Understated confidence throughout. Better for productivity, SaaS, professional tools.

### Combinatorial Math

1 validated concept x 5 hooks x 4 avatars x 3 lengths x 2 registers = **120 possible variations**

In practice, produce 20-25 per production cycle (the highest-priority combinations), not all 120.

## Cross-Platform Unified Generation

Generate platform-native scripts for TikTok video, Instagram Reels, YouTube Shorts, and Facebook Reels from a single Claude session using one underlying brief. The same content concept gets calibrated to each platform's algorithmic dynamics rather than written for one platform and crudely adapted for the others.

### The 3-input brief

Every cross-platform Claude session loads three inputs upfront:

1. **Audience insight statement**: vivid emotional portrait built from real customer language (Reddit, sales calls, review mining), not a demographic description
2. **Product brief**: unique mechanism articulation, specific proof points with timelines and numbers, price/offer structure
3. **Structural reference**: annotated transcript from a competitor reel running 30+ days, with structural labels at hook, pain, mechanism bridge, product introduction, proof, and CTA

The brief loads once per session. Multiple platform-specific generations run within that session.

**Full brief structure:** See `references/cross-platform-brief-structure.md`.

### The 4 platform calibration variables

Each platform calibrates differently across:

1. **Duration**: TikTok 15-25s, IG Reels 18-30s, YT Shorts 30-45s, FB Reels 25-45s
2. **CTA architecture**: TikTok DM share, IG saves + comment funnel, YT subscribe, FB substantive comment
3. **Caption length and style**: TikTok aggressive, IG Stories aesthetic, YT title+description+thumbnail, FB 3-5x longer
4. **Audio approach**: TikTok trending overlay or voiceover-only, IG/YT/FB voiceover-only with calmer pacing

**Full calibration table:** See `references/platform-calibration-variables.md`.

### Master prompts

Six prompts cover all generation needs:

- TikTok video master prompt
- Instagram Reels master prompt
- YouTube Shorts master prompt (includes title, description, thumbnail concept)
- Facebook Reels master prompt (older audience calibration, longer captions)
- Cross-platform parallel generation (all 4 platforms in one Claude session)
- Whole-script variation from a winner (5-6 lite variations per platform)

**Full prompt template:** See `templates/master-prompts-by-platform.md`.

### Cross-platform intelligence layer (weekly)

Every Monday, run the unified intelligence session that synthesizes performance data across all 4 platforms into a single analysis. Identifies cross-platform patterns (hook structures winning across platforms, saturation signals appearing on one platform before others, audience portability between platforms) that fragmented single-platform analysis misses.

The intelligence session updates the 3-input brief and feeds the next week's production priorities.

**Full intelligence framework:** See `references/cross-platform-intelligence.md`.

### Why unified beats fragmented

Operators running separate Claude sessions per platform do 4x the strategic work to produce the same content variety, and miss the cross-platform patterns that only emerge when data flows through a single analytical pipeline.

A single 60-minute Claude session running the cross-platform parallel generation can produce platform-specific scripts for 5-6 different content concepts across all 4 platforms (20-24 platform-specific scripts ready for production). The same volume from single-platform sessions takes 4x the time and produces shallower intelligence.

## Winner Variation Pipeline (Component-Level Multiplication)

The 4-axis matrix above runs at the concept level. The Winner Variation Pipeline runs at the component level. It turns 1 validated winning reel into 1,000+ tested variations through a hook + body + CTA component swap with explicit compatibility filtering.

### When to trigger

A reel earns the variation pipeline when:
- Hook rate clears 50%
- Hold rate stays strong behind the hook
- The signal is real, not random batch noise

Reels that have not cleared these thresholds do not get variation-pipelined. Multiplying weak content produces more weak content.

### The 8-step workflow

1. Strip hook, body, and CTA out as isolated assets
2. Generate 25 hook variations across 4 categories (Specificity, False Opening, Result Specificity, Identity)
3. Produce all 25 hooks in Seedance 2.0 with locked character + scene baseline
4. Generate 12-15 body variations across 6 emotional framings (Frustration, Wasted Money, Social Comparison, Identity Threat, Failure Stack, Unfair Comparison)
5. Generate 10-12 CTA variations across 3 action types (Comment Funnel, DM Share, Save Prompt)
6. Run the compatibility matrix: tag every component, filter incompatible pairings
7. Filter 3,000 theoretical combinations to roughly 1,000 coherent assembled reels
8. Deploy first batch (40-50) with component-level performance tracking

### The 8-week compounding loop

Run the pipeline as an 8-week loop. Retire underperforming components across the library. Expand winning components with new variations. After 8 weeks the library has more validated audience intelligence than most operators accumulate in a year.

## Daily Production Schedule

Separate creative decisions from mechanical production. Mixing them is the primary cause of production inefficiency.

### Block 1: Creative Decisions (90 minutes)

All creative judgment for the day happens here:
- Which concepts are being varied today?
- Which hook variations are being written?
- Which avatar profiles are being selected?
- Which length variants are being prepared?

**Output:** 20-25 complete script variants ready for voice-over, organized by concept and variation axis.

### Block 2: Voice-Over Generation (60 minutes)

Batch-generate all voice-overs using `elevenlabs` speech-to-speech with UGC presets (stability 0.35, similarity 0.50). Submit all scripts in rapid sequence.

**Output:** 20-25 completed voice-over files, labeled by concept, variation axis, and target account.

### Block 3: Video Generation (90 minutes)

Batch-generate all videos using `jogg-ai`. Upload each avatar image + voice-over file and queue renders simultaneously.

**Output:** 20-25 completed video files ready for captions.

### Block 4: Captions, Formatting, Scheduling (60 minutes)

Add captions (Instagram native or Captions app). Format for destination platform. Schedule via Later or Buffer for optimal posting times per account's target time zone.

**Output:** 20-25 fully produced, captioned, formatted, and scheduled pieces of content.

**Total:** 4 blocks, 5.5 hours, 20-25 pieces per cycle. Running 2 cycles/day = 40-50 pieces.

## Multi-Account Portfolio System

### 3-Tier Architecture

**Tier 1: Flagship (2-3 accounts)**
- Highest-authority accounts with 90+ days consistent posting
- Receive highest-performing content first
- Primary vehicle for paid amplification
- Requirements: >3.5% engagement rate, thematic consistency, genuine persona-matched followers

**Tier 2: Building (8-15 accounts)**
- Primary volume engine. 3-5 posts/day per account.
- Receive validated hook/avatar variations from testing rotation
- 20-30 genuine niche interactions per day per account
- Weekly review: watch-through rate, save rate, CTR per account

**Tier 3: New/Warmth Period (5-8 accounts)**
- 28-day graduated warmth protocol before full-volume posting
- Always maintain 5-8 accounts in warmth to replace retired accounts

### Account Warmth Protocol (28 Days)

| Period | Posts/Day | Engagements/Day | Follows/Day | Content Type |
|--------|-----------|-----------------|-------------|--------------|
| Day 1-7 | 1 | 10 | 20-30 | Proven content from Tier 1/2 |
| Day 8-14 | 2 | 15 | 20-30 | Proven + 1 new original/week |
| Day 15-21 | 3 | 20 | - | Introduce variation testing |
| Day 22-28 | 3-5 | 25-30 | - | Full Tier 2 volume. Graduated. |

**Why gradual:** The graduated behavioral pattern is the signal the algorithm associates with genuine organic accounts. Full volume from Day 1 looks like a content farm.

## Weekly Optimization Review (Every Monday)

### Step 1: Identify Top 5 and Bottom 5

Rank all content from the previous week by click-through rate. For each of the top 5 and bottom 5, document:
- Hook category
- Avatar type
- Script concept
- Length
- Platform
- Account posted on

### Step 2: Extract the Pattern

What do the top 5 have in common that the bottom 5 don't? This pattern insight is the single most valuable output of the weekly review.

### Step 3: Update Variation Matrix

- Which variables have enough data to draw conclusions?
- Which need more test volume?
- Which new combinations does this week's data suggest?

### Step 4: Set Next Week's Production Priorities

Based on the pattern insight, decide:
- Which hook categories to double down on
- Which avatar types to prioritize
- Which concepts to retire
- Which new concepts to test

### North Star Metric

**Revenue per piece of content posted.** Track weekly.

```
Revenue Per Content = Total Weekly Revenue / Total Pieces Posted
```

- **Increasing:** System is learning. Keep current approach.
- **Flat:** Optimization stalled. Review quiz conversion, email sequences, hook quality.
- **Declining:** Something broke. Diagnose: content quality, funnel, or email layer.
